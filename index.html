A web based interface to look at your subversion history. Provides the following graphs

  • Important words used in commits based on term frequency
  • Punch card (Day vs Hour of day, cell value = number of commits)
  • Monthwise number of commits
  • Monthwise files modified
  • A co-author graph (People who have worked on similar files more number of times are closer)

If the user is 'all' at certain path in the repo, the following extra graph is displayed

  • Pie chart of the contribution by the user

Pre-Requisites

  • Apache
  • PHP
  • MYSQL
  • Internet access for generating charts

How to run:

Step1: Populating data

  • Create a verbose svn log of your repository and call it 'svn_log_verbose_to_add.xml' eg: svn log -v --xml > svn_log_verbose_to_add.xml

  • Create a schema in mysql called svn_analysis Import the queries in svn_analysis.sql. This will create couple of tables

  • Modify the mysql host, username and password in the file parse.php. The user account used must have write permissions to svn_analysis schema.

  • run parse.php. This will populate the data from svn_log_verbose_to_add.xml to svn_analysis schema. Depending on the number of commits, this might take a while.

Step2: Running

  • Modify the mysql host, username and password in the file data/db.php. The user account used must have read permissions to svn_analysis schema created.

  • Whole folder of svn_analysis should be under doc root for apache. Create an alias in apache to point /svn_analysis to this directory. index.php is the main page.

Improvements

  • Incremental svn logs addition
  • Use a JavaScript based charting library
  • Read the db credentials from a properties file for both data store and display
  • Use a word cloud for display of commit words. Also use stemmers and a better stop word removal.
